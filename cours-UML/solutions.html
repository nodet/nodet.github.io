2. Identification des acteurs et de cas simples

Le système considéré est une station service de distribution de carburant. Les clients se servent en carburant, et le pompiste remplit les cuves.

  1. Le client se sert de la façon suivante : il prend un pistolet accroché à une pompe et appuie sur la gachette pour faire couler le carburant. Qui est l’acteur dans ce cas d’utilisation ? Le client, le pistolet ou la pompe ?

C’est le client. Un acteur est toujours extérieur au système étudié.

  1. Pierre est pompiste. De temps en temps, il vient à la station pour remplir le réservoir de sa voiture. Pour modéliser cette activité, faut-il un nouvel acteur ?

Non. On considère le rôle de chaque intervenant, et non pas son identité. Le fait que Pierre soit pompiste ne change rien à ce qui se passe lorsqu’il se sert en carburant. Donc il n’y a pas de nouveau rôle.

  1. Certains pompistes sont qualifiés pour effectuer des opérations que tous les pompistes ne peuvent pas faire, comme des opérations de maintenance. Ils sont donc réparateurs en plus d’être pompistes. Comment modéliser cela ?

Le Technicien est une spécialisation du Pompiste, puisqu’il sait faire tout ce que sait faire le pompiste, plus autre chose.

4. Exemples de cas d’utilisation : la bibliothèque

Le système étudié est le poste de travail muni du matériel et du logiciel de gestion des emprunts de documents dans une bibliothèque.

4.1. Cas d’utilisation : identification d’un utilisateur

Liste des acteurs

  • Le bibliothécaire

Préconditions

  • Le bibliothécaire est identifié

Scénario nominal

  • Le bibliothécaire scanne la carte de l’utilisateur à l’aide d’une douchette.

  • Le logiciel présente au bibliothécaire les informations connues sur le porteur de la carte, et un signal sonore de type 'validation' est émis.

  • Ces informations sont : nom et prénom, adresse, liste des documents empruntés.

  • La liste des documents comprend les informations suivantes pour chaque document : type (livre, CD, ou DVD), titre, date de retour maxi.

  • Si la date de retour maxi est égale ou antérieure à la date du jour, le document apparaît avec un marquage spécifique de type 'interdiction'.

  • Si la date de retour maxi est postérieure à la date du jour de 7 jours ou moins, le document apparaît avec un marquage spécifique de type 'attention'.

Enchainement d’erreur

  • Si la carte est inconnue ou illisible, un message est affiché et un signal sonore est émis.

Post-condition

  • Le système est dans l’état "Utilisateur identifié"

4.2. Cas d’utilisation : scan d’un document

Liste des acteurs

  • Le bibliothécaire

Préconditions

  • Le bibliothécaire est identifié

Scénario nominal

  • Le bibliothécaire scanne le code barre du document à l’aide de la douchette.

  • Le système affiche les informations concernant ce document : type, auteur, titre, date de dernier emprunt.

Enchainements d’erreur

  • Si le code scanné ne correspond pas à un document en stock, un signal sonore et visuel est émis, un message "Document inconnu" est affiché.

Post-conditions

Aucune post-condition. Ce cas d’utilisation ne modifie aucune information dans le système. Il n’y a donc pas lieu de vérifier quoi que ce soit qui n’a pas été vérifié avant.

4.3. Cas d’utilisation : emprunt d’un document

Liste des acteurs

  • Le bibliothécaire

Préconditions

  • Un utilisateur est identifié

Enchaînement nominal

  • Scan d’un document se termine sans erreur

  • La liste des documents empruntés déjà présente sur l’écran est complétée du document scanné.

Enchaînement d’erreur

  • Si le document est marqué "emprunté", un signal sonore et visuel est émis, un message "Document déjà emprunté" est affiché.

  • Si le document est marqué "réservé" et que l’utilisateur n’est pas celui pour lequel le document est réservé, un signal sonore et visuel est émis, un message "Document réservé" est affiché.

  • Si le nombre de documents déjà empruntés par l’utilisateur est supérieur ou égal à la limite, un signal sonore et visuel est émis, et un message "Trop de document empruntés" est affiché.

Post-conditions

  • Le document est marqué "emprunté" dans la base de donnée du système avec une référence vers l’utilisateur.

Exigences non fonctionnelles

  • Tous les affichages du système seront étudiés pour être lisibles en niveaux de gris aussi bien qu’en couleur.

11. Python : les Mangenombres

class MangeNombres:

  def __init__(self, p):
    self.p = p                           (1)
    print(p)                             (2)
    self.suivant = None                  (3)

  def mange(self, q):                    (4)
    if (q % self.p == 0):
      return                             (5)
    if self.suivant:
      self.suivant.mange(q)              (6)
    else:
      self.suivant = MangeNombres(q)     (7)

N = 2
m = MangeNombres(N)
for i in range(N+1, 20):
  m.mange(i)

  1. Tous les multiples de p seront mangés

  2. p étant premier, on l’affiche

  3. On créera le MangeNombres suivant quand on nous donnera un nombre non multiple de p à manger

  4. La fonction qui 'mange' le nombre ou le passe au suivant

  5. Si q est un multiple de p, on le mange et c’est fini pour lui

  6. Sinon on le donne au MangeNombres suivant s’il existe

  7. q n’a été mangé ni par ce MangeNombres, ni par aucun de ses prédécesseurs. Donc q est premier, et créer le MangeNombres suivant va (1) afficher q et (2) s’assurer que ses multiples seront mangés

12. Python : Vehicles

class Vehicle:

  def __init__(self, name, capacity):
    self.name = name
    self.capacity = capacity

  def fare(self):
    return self.capacity * 100

  def __repr__(self):
    return f"{self.name}: {self.fare()}"


class Car(Vehicle):

  def __init__(self, name):
    super().__init__(name, 5)


class Bus(Vehicle):

  def fare(self):
    return super().fare() * 1.1


v = Car("VW Touran")
print(v)

v = Bus("School Volvo", 50)
print(v)

Le résultat de ce programme est:

VW Touran: 500
School Volvo: 5500.0

13. Python : A budget app

class Category:
  """
  Records a budget, a list of expenses.
  Budget can be updated later
  """

  def __init__(self, name, remaining):
    self.name = name
    self.expenses = []
    self.remaining = remaining

  def __repr__(self):
    return (self.name + '('
      + str(self.remaining) + ' -- '
      + str(self.sum_of_expenses()) + ')')

  def add_expense(self, amount):
    self.expenses.append(amount)
    self.remaining -= amount

  def sum_of_expenses(self):
    return sum(self.expenses)

  def add_to_budget(self, amount):
    self.remaining += amount


class Budget:
  """
  This class allows to record expenses and
  categorize them into various types of
  expenses
  """

  def __init__(self, total):
    self.initial_budget = total
    self.total = total
    self.categories = {
      # Each category will be a name and a list
      # of amounts
    }

  def sum_of_expenses(self):
    somme = 0
    for name, category in self.categories.items():
      somme += category.sum_of_expenses()
    return somme

  def print_status(self):
    print(f'Initial budget: {self.initial_budget}')
    print(f'Remaining budget: {self.total}')
    somme = self.sum_of_expenses()
    print(f'Sum of expenses: {somme}')
    for key, val in self.categories.items():
      print(val)
    print()

  def add_category(self, name, amount):
    assert(name not in self.categories)
    self.categories[name] = Category(name, amount)

  def add_expense(self, cat, amount):
    assert(cat in self.categories)
    self.categories[cat].add_expense(amount)
    self.total -= amount
    assert(self.total + self.sum_of_expenses() == self.initial_budget)

  def balance(self, source_category, destination_category, amount):
    assert(source_category in self.categories)
    assert(destination_category in self.categories)
    self.categories[source_category].add_to_budget(-amount)
    self.categories[destination_category].add_to_budget(amount)

  def proportions(self):
    total = self.sum_of_expenses()
    l = []
    for f, c in self.categories.items():
      this_cat = c.sum_of_expenses()
      l.append((f, this_cat / total))
    return l

b = Budget(100)
b.print_status()
b.add_category('Food', 20)
b.add_category('Transport', 10)
b.print_status()
b.add_expense('Food', 1.4)
b.add_expense('Transport', 4.6)
b.print_status()
b.balance('Food', 'Transport', 2.8)
b.print_status()
print(b.proportions())
